1. Abstract

VoteSecure is a comprehensive web-based Electronic Voting Machine (EVM) system developed to revolutionize the voting process. This project addresses critical issues in traditional voting such as long queues, potential fraud, lack of transparency, and high logistical costs.

The system provides end-to-end election management including voter registration with secure password hashing (bcrypt), electronic vote casting with one-person-one-vote enforcement, real-time election controls by administrators, and interactive result visualization with bar charts and winner declarations.

Key technical features include PHP 8+ backend with MySQL 10.4 database running on XAMPP stack, responsive glassmorphism UI, SQL injection prevention through parameterized queries and escaping, session-based authentication, and comprehensive validation. The system supports complete election lifecycle: setup → registration → voting → results → reset.

Successfully tested with 100% pass rate across all modules. Scalable for educational institutions and small-scale elections. Future enhancements include blockchain integration and multi-constituency support.

6. Implementation Details

6.1 Critical Functions Explained

6.1.1 Database Connection (db.php)

Purpose: Centralized DB connection with error recovery and session initialization. Port 3307 specific for XAMPP MySQL.

6.1.2 Voter Registration (register.php)

$hashed = password_hash($pass, PASSWORD_DEFAULT);
$name_esc = mysqli_real_escape_string($conn, $name);
$email_esc = mysqli_real_escape_string($conn, $email);
$q = mysqli_query($conn, "INSERT INTO users(name,email,password) VALUES('$name_esc','$email_esc','$hashed')");

Security: Bcrypt hashing (60+ char salts), SQL escaping, duplicate email check. Minimum 4-char password.

6.1.3 Vote Casting Logic (evm.php)

// Triple validation before vote
$voted_already = ($user['voted'] == 1);
if(isset($_POST['vote']) && !$voted_already && $election_status == 'ongoing'){
    $candidate_id = intval($_POST['candidate_id']); // Sanitize
    // Atomic updates
    mysqli_query($conn, "UPDATE candidates SET votes = votes + 1 WHERE id=$candidate_id");
    mysqli_query($conn, "UPDATE users SET voted = 1 WHERE id=$user_id");
    header("Location: vote_success.php");
}

Integrity: Checks user vote status, election status, candidate existence. Uses INT casting and atomic DB updates.

6.1.4 Admin Election Control (admin_dashboard.php)

// State machine transitions
if(isset($_POST['start_election'])){
    mysqli_query($conn, "UPDATE election_status SET status='ongoing'");
}
if(isset($_POST['end_election'])){
    mysqli_query($conn, "UPDATE election_status SET status='ended'");
}
// Max 4 candidates enforcement
$count = mysqli_fetch_assoc(mysqli_query($conn, "SELECT COUNT(*) as cnt FROM candidates"))['cnt'];
if($count >= 4) $error = "Maximum 4 candidates";

State Machine: not_started → ongoing → ended → result_declared → reset. Prevents invalid transitions.
